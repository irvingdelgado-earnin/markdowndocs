# Review Rubric

## Critical Issues
Use for:
- logic bugs
- crashes
- race conditions
- architecture breakages
- unsafe threading
- missing handling for important edge cases

## Major Issues
Use for:
- significant test gaps
- dependency misuse
- misplaced responsibilities
- unsafe patterns likely to cause future bugs
- confusing architectural choices
- missing analytics or rollout safeguards for material behavior changes

## Moderate Issues
Use for:
- avoidable complexity
- duplication
- weak naming
- maintainability concerns

## Minor Issues or Nits
Use for:
- readability polish
- stylistic cleanup
- small API ergonomics suggestions

## Comment Quality
Only include comments that are:
- specific
- defensible
- actionable
- worth a human reviewer posting
