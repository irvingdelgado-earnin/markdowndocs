# Analytics Event Patterns

A generic link-oriented offer usually needs analytics across:
- multiple entry points such as home promo card, discovery icon, discovery list, new promo placements, and post-cashout
- multiple states such as not enrolled versus enrolled
- multiple surfaces such as preview card, expanded card or landing page, web view, and learn more or terms screens
- a mix of legacy reused events and offer-specific attributes

## Common attribute patterns
- funnelType
- screenContext
- productName
- cardText
- placementId
- offerId
- offerRank
- buttonContext

## Typical checkpoints
- promo viewed
- promo tapped
- landing page viewed
- primary CTA tapped
- learn more or terms tapped when applicable
- web view opened
