# UI Rules

## Design System First
- Prefer existing design system components over custom UI components.
- Reuse established patterns in the module before introducing new ones.

## Weak Match Handling
- If no strong design system match exists, do not silently invent bespoke UI.
- Call out the mismatch explicitly.
- Present the tradeoff and ask for review.

## Output Expectations
When UI is involved, say:
- which design system component is being used
- where the match is weak, if applicable
- what needs design confirmation, if any
